# ADR 002: Estrategia de Ramificación
**Fecha:** 25 de Abril de 2026 [cite: 39]
**Estado:** Aceptado

## Decisión
Se adopta el uso de la rama `feature/docs` para todo el trabajo de manuales técnicos, protegiendo la integridad de `main`.

## Consecuencias
[cite_start]Se evita el "código invisible" al asegurar que cada cambio en la documentación sea revisado mediante Pull Requests[cite: 6].
